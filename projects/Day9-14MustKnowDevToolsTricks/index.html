Console Dev Tricks

Here are 13 console tricks to throw in your er'y day dev kit.

Tip: Copy and Paste the input into your own Dev Tools Console <F12>

Regular console.log()

Using plain old console.log() with a simple supplied string.

Command

console.log('hello');

Outputs

hello

Interpolated console.log()

Supplying a variable to the console.log() to use in place of the placeholder.

Command

console.log('Hello there %s!', 'Steve');

Output

Hello there Steve!

Styling console output using %c

We can also style the output we send to the console. Anything that follows the %c will be styled according to the styles provided in the second parameter.

Command

console.log('Apply now and %cget your dream job!', 'font-size: 1.5em; background: blue; color: white');

Output

Apply now and get your dream job!

Warnings using console.warn()

We can issue a warning to the console as well. It gives a nice icon and an expanding arrow with information about where the warning was called.

Command

console.warn('Careful now!');

Output

Careful now!

Errors using console.error()

We can also send errors to the console. Just like console.warn() this provides an appropriate icon and addition information.

Command

console.error('Well, now you've gone and done it!!);

Output

Well, now you've gone and done it!!

Logging info using console.info()

Sometimes we just want to let our console viewer know something. This is a great reason to use console.info(). There really isn't much of a difference between info and log beyond the symantics, and an info icon.

Command

console.info('Now, here is something interesting!);

Output

Now, here is something interesting!

Clearing the console using console.clear()

If you're looking to hide all those errors and warnings, console.clear() is your ticket! It takes no parameters, and will (attempt to) clear the console window, however, most browsers have a "preserve log" setting, that overrides this. Better to deal with the errors and warnings. ;)

Command

console.clear();

Output

Console was cleared

Testing with the console using console.assert()

You can use console.assert() to run a quick test. This method takes 2 parameters; the test case, and what to display if the test fails. In our example below, the first test has failed, and we see our 'failed' message. The second assert passes and we get nothing (except 'undefined').

Command

let assertElement = document.querySelector('.select-me');
console.assert(assertElement.classList.contains('nope'), 'Sorry, that failed.');
console.assert(assertElement.classList.contains('yup'), 'Sorry, that failed.');

Output

Assertion failed: Sorry, that failed.
undefined

Viewing DOM elements with console.dir()

We can view any DOM element we have a reference to using console.log(element) and it shows us the element. Sometimes however, we need the properties of the element. That's where console.dir(element) comes in. Our first line of output comes from console.log(showMe) and is the actual DOM element. The second bunch of output is from console.dir(showMe). As you can see it give us a nice list of the node's properties.

Command

let showMe = document.querySelector('.select-me');
console.log(showMe);
console.dir(showMe);

Output

<span class='select-me yup'></span>

span.select-me.yup
    accessKey : ""
    assignedSlot : null
   attributeStyleMap : StylePropertyMap {size: 0}
    ...

Grouping output with console.group() and console.groupEnd()

When we have a bunch of related output on the console, we are able to group it together using console.group(). The group is opened and given a name. You can close that group by calling console.groupEnd() and supplying the same name. Tip: If you want to keep your console clean (and of course you do), use console.groupCollapsed() instead.

Command

const groupDogs = [{ name: 'Salty', age: 2 }, { name: 'Molly', age: 14 }];

groupDogs.forEach(dog => {
   console.group(`${dog.name}`);
    console.log(dog);
    console.log(`Age in dog years: ${dog.age * 7}`);
   console.groupEnd(`${dog.name}`);
});

Output

Salty
    {name: "Salty", age: 2}
    Age in dog years: 14
Molly
    {name: "Molly", age: 14}
    Age in dog years: 98

Counting number of logs with console.count() and console.countEnd()..

This method will count the number of times a specified output has been logged to the console.

Command

console.count('Salty');
console.count('Salty');
console.count('Salty');
console.count('Salty');
console.count('Molly');
console.count('Molly');
console.count('Salty');
console.count('Salty');

Output

Salty: 1
Salty: 2
Salty: 3
Salty: 4
Molly: 1
Molly: 2
Salty: 5
Salty: 6

Timing with console.time() and console.timeEnd()..

This method allows you to start and stop a timer (or number of timers) using a string in the parameter list as a reference. Using the same string you used to start the timer, you can call console.timeEnd(str) to stop the timer. It will then log the results to the console.

Command

console.time('fetchdata');
fetch('https:api.github.com/users/patrickfranz')
   .then(resp => resp.json())
   .then(data => {
      console.timeEnd('fetchdata')
    });

Output

fetchdata: 564.93603515625ms

Display data in a table using console.table()

If you have an iterable collection such as an Array, it often easier to see what is going on with it when it's formatted in a table.

Command

const doglist = [{ name: 'Salty', age: 2 }, { name: 'Molly', age: 14 }];
console.table(doglist);

Output

(index) name age
0 Salty 2
1 Molly 14
